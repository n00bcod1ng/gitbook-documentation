# Ticket #4 – New Employee Laptop & Account Setup

**Reported by:** Lisa Patel – Operations **Priority:** Medium **Category:** New Employee Setup **Type:** Service Request **Status:** Closed



#### Request

The Operations manager submitted a service request for a new employee, David Kim, starting Monday in the Operations department. The request included a laptop imaged with the standard corporate build, an Active Directory account with the appropriate group memberships, an email mailbox, and access to the Operations shared drive. Hardware needed to be ready for pickup by Friday end of day.

#### Fulfillment

Unlike the previous incidents, this ticket was a service request — a planned provisioning task rather than something broken. A laptop was pulled from inventory and the asset management record was updated with the assigned user and deployment date. The device was imaged with the standard corporate Windows 11 build and fully patched. David Kim's Active Directory account was created in the Operations OU following the standard naming convention, added to the Operations security group for shared drive access, and a mailbox was provisioned and verified with a test message. The laptop was joined to the domain, Group Policy application was confirmed at first login, and a full test login verified shared drive access, email, and standard applications. A temporary password was set with "User must change password at next logon" enabled, and the requester was notified the laptop was ready ahead of the Friday deadline.





**Step-by-step fulfillment summary**

1. Pulled a laptop from inventory and updated the asset management record.
2. Imaged the device with the standard corporate Windows 11 build and applied all updates.
3. Created the new hire's Active Directory account in the Operations OU.
4. Added the account to the Operations security group for shared drive access.
5. Provisioned the user's mailbox and verified mail flow with a test message.
6. Joined the laptop to the domain and confirmed Group Policy applied at first login.
7. Performed a full test login — verified shared drive, email, and applications.
8. Set a temporary password with forced change at next logon.
9. Notified the requester that hardware was ready ahead of the deadline.
